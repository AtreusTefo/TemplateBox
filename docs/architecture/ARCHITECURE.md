📁 The Optimal Folder StructureTo ensure a seamless, frictionless drag-and-drop deployment on Netlify, the project directory must be kept flat, modular, and organized. Save your files in a single root folder configured exactly like this:

templatebox/
│
├── index.html          # Homepage (Template Catalog & Category Filter)
├── loading.html        # 10-Second Ad Intermediary Page (Timer, Banners, Social Bar, inline countdown fallback)
├── resume.html         # CV/Resume Builder Editor
├── poster.html         # Poster & Canvas Creator Editor
├── mockup.html         # Print-on-Demand Mockup Generator Editor
├── blog.html           # Public Blog Index
├── post.html           # Single Blog Post Renderer (?slug=)
├── admin.html          # Private Blog Authoring Panel (noindex, robots-disallowed)
├── privacy.html        # Privacy Policy & Data Processing Compliance Notice
├── serve.json          # Local Static-Server Config (disables clean-URL query stripping)
│
├── css/
│   └── style.css       # Unified Global Stylesheet (Fabric Film Studio Theme)
│
├── js/
│   ├── app.js           # Shared App Logic (Sanitization, Catalog, Loading-Page Countdown, Editor Route Whitelist)
│   ├── resume.js         # CV Builder Core Logic (localStorage, jsPDF)
│   ├── poster.js         # Poster Creator Core Logic (HTML5 Canvas Drawing)
│   ├── mockup.js         # Mockup Generator Core Logic (HTML5 Canvas Drawing)
│   ├── blog.js           # Shared Blog Library (Content Model, XSS-Safe Rendering, Ad Zone Registry)
│   ├── blog-data.js      # Static Blog Post Database (generated by admin.html, deployed as an asset)
│   └── admin.js          # Blog Admin Panel Logic
│
└── assets/
    └── logo.png        # Brand Logo

Adding a fourth editor: register it in `EDITOR_ROUTES` in `js/app.js` AND in the mirror route table inside the inline countdown fallback in `loading.html` — the two must stay in sync, see `docs/error-fixes/LOCAL_SERVE_CLEAN_URL_DROPS_TARGET_QUERY.md` and the mockup-generator implementation notes in `docs/implementation/` for the full extension pattern.
